Algorithms

Human kind has been working on problems for centuries. So, any problem you are trying to solve (or a sub problem you are facing while solving a large problem) might not be totally new. There is a high possibility that someone has already discovered an efficient way to solve similar type of problem efficiently. We better use them. Why? Let’s take an example:

pseudocode 1:
a = 5
n = 100
while (n != 0)
  a++
  n--
print (a)
pseudocode 2:
a = 5
n = 100
a = a + n
print (a)


Both of the pseudocodes will print the same result 105. But which one is faster? The 2nd one, right? But some of you may argue that adding 1 to a, 100 times don’t even matter for our advanced processors. A 2 GHz processor can easily handle 2 billion operations per second. But what if the value of n is 200 billion? It would take 100 seconds for a 2 GHz processor to execute pseudocode 1 while it takes 10^-9 seconds to execute the 2nd one.

Computers can perform only a limited number of operations per second. In real life problems we always work with large numbers. Think about google maps (Trillions of roads & routes), facebook (more than 2.3 billions of users), medical diagnosis studies (human genome, neurons etc)… these can be some real life examples of use of large numbers. So, learning algorithms is necessary.
